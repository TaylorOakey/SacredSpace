## I. The Three Tiers

Every sigil in SacredSpace operates across three tiers of meaning:

```
TIER 1 — NAVIGATION (WHERE)
  [Dimension Glyph] — Which domain of the OS to address
  Controlled by: Resonance Economy (does the caster have access?)

TIER 2 — OPERATION (WHAT)
  [Root Sigil(s)] — Which fundamental action to perform
  Governed by: Grammar Rules (how do sigils compose?)

TIER 3 — SIGIL STRING (HOW)
  [Dimension Glyph] + [Root Sigil(s)] + [Grammar Affixes]
  The full composed invocation — an executable intention.
```

Each tier is independent. A Root Sigil works the same way whether cast in VAULT or FOREST. A Dimension Glyph is always the same symbol regardless of which Root Sigil it gates. The stack is **orthogonal by design**.

### Dimension Glyph Rules

1. **A Dimension Glyph opens access to its pillar** — it does not specify what to do there. That is Tier 2's responsibility.
2. **Multiple dimension glyphs can be chained** to cast a cross-dimensional sigil.
3. **Resonance cost** is assessed per dimension accessed (see Section VII).
4. **All nine glyphs are always available** — resonance determines whether a cast succeeds, not whether the glyph is known.

## III. Tier 2 — Operation Layer: Root Sigils

*Source: Sigil Engine (Gemini-era, 2025-12-11)*

Six Root Sigils are the **operation layer** — the fundamental actions that can be performed within any dimension. They are the "alphabet of gods" for SacredSpace.

| # | Glyph | Name | Function | Vibe | Gesture |
|---|-------|------|----------|------|---------|
| 1 | ⟁ | GATEWAY | Opening / Orientation — portals, navigation, linking | GPS meets intuition | Draw triangle with vertical axis |
| 2 | ○• | MOTE | Memory Particle — notes, tags, fragments, journaling | A seed of meaning | Draw circle with orbiting dot |
| 3 | ✧ | QUEST | Path & Progression — tasks, roadmaps, XP, rituals | A map that is also a heartbeat | Draw four-point star with curve |
| 4 | ⚒ | FORGE | Creation & Crafting — templates, generators, builds | The act of making real | Draw intersecting hammers / crescents |
| 5 | Ϟ | MAESTRO | Resonance & Rhythm — sound, emotion, learning | The OS hearing itself | Draw lightning-bent eighth note |
| 6 | ✦ | LANTERN | Illumination & Insight — analytics, guidance, wisdom | Presence meeting clarity | Draw diamond with flame inside |

### Root Sigil Rules

Root sigils follow three grammar rules inherited from the Sigil Engine:

**Rule 1 — Shape Carries Purpose:**
| Shape | Meaning |
|-------|---------|
| Curved (○•, Ϟ) | Receptive, intuitive, storing |
| Angular (✧, ⚒) | Active, directive, invoking |
| Hybrid (✦, ⟁) | Transformation or transition |

**Rule 2 — Stacked Sigils Create Operations:**
When two or more root sigils are composed, the resulting meaning is an operation:

| Composition | Meaning |
|-------------|---------|
| ⟁ + ✧ | Begin a new Quest at the Gateway |
| ○• + ⚒ | Turn a memory into a crafted artifact |
| Ϟ + ✦ | Illuminate the emotion within a sound |
| ⚒ + ✧ | Forge a new path / create a quest |
| ✦ + ⟁ | Illuminate the way forward |
| ○• + ✧ | Remember a journey / log progress |

**Rule 3 — Active Only When Paired With a Gesture:**
A root sigil on its own is a noun. It becomes a verb when manifested:

| Gesture | Effect |
|---------|--------|
| Draw (full invocation) | Full cast — complete operation |
| Tap (quick access) | Quick peek — single result |
| Hold (amplify) | Deep dive — rich context |
| Swipe (morph/transform) | Shift to related context |
| Circle (loop/repeat) | Repeat last operation |

## IV. Tier 3 — Sigil String: Composition

A **Sigil String** is the fully composed invocation combining all three tiers:

```
[DIMENSION GLYPH] + [ROOT SIGIL(s)] + [GRAMMAR AFFIXES]
```

### Composition Syntax

```
<dimension>:<operation> [<params>]
```

Where:
- `<dimension>` = one or more dimension glyphs (∞, ◊, ∆, ⊙, ≈, ♦, ⊗, ⊜, Λ)
- `<operation>` = one or more root sigils (⚙ grammar-composed)
- `<params>` = optional affixes (target, intensity, scope)

### Examples

| Sigil String | Meaning | Effect |
|-------------|---------|--------|
| `∆:⚒` | FOREST:FORGE | Generate a new neural graph extraction in the Forest |
| `⊙:✦` | CODEX:LANTERN | Illuminate insight from the Codex |
| `∞:○•` | VAULT:MOTE | Store a memory mote from the Vault |
| `Λ:✧` | PATH:QUEST | Begin a new learning quest on the Path |
| `⊗:Ϟ` | SOCIAL:MAESTRO | Resonate social content for emotional tuning |
| `∆+⊙:⚒+✦` | FOREST+CODEX:FORGE+LANTERN | Cross-dimensional insight synthesis across Forest and Codex |
| `♦:✧+⚒` | AGENT:QUEST+FORGE | Forge a new agent capability as a quest |

### Grammar Affixes

| Affix | Symbol | Meaning |
|-------|--------|---------|
| Scope | `!` | Broadcast — all dimensions |
| Intensity | `+` | Amplified — deep context |
| Repeat | `~` | Loop — continuous operation |
| Transform | `>` | Morph to related context |
| Pin | `*` | Persist as mote |
| Limit | `#N` | Top N results |

The five Weaver Spells from the Grimoire are now understood as **macros** — named shorthand for common Root Sigil compositions that invoke specific ICARIS agents.

| **ASHER.SHADOW()** | `asher_shadow` | `✦:⚒` | ASHER | Apply Lantern to Forge — illuminate what is hidden |
| **SCRIBE.RECORD()** | `scribe_record` | `○•:⚒` | SCRIBE | Turn a Mote into a crafted record |

### How Macros Work

When a macro is cast, the engine expands it into its full Root Sigil composition, then resolves each component:

```
User casts:  AURORA.WEAVE() with query: "trace sigil magic"
Engine sees: ∆+⊙:✦+╻ [FOREST+CODEX : LANTERN+GATEWAY]
Resolves to: Query FOREST (ChromaDB) + CODEX (ChromaDB)
             Apply LANTERN (Ollama synthesis)
             Open GATEWAY to receive results
```

### Extension — Custom Macros

Users can define their own macros via the Sigil Terminal:

```sigil

macro GROVE.COUNCIL() = ◊:✦  // Illuminate council insights
```

## VI. The 5-Layer Engine

Every sigil cast passes through five layers of execution. This is the **operational spine** of the Sigil Engine:

```
┌─────────────────────────────────────────────────────────────┐
│                      SIGIL ENGINE                            │
│                                                              │
│  INPUT ──→ GRAMMAR ──→ RESOLUTION ──→ MANIFESTATION ──→ ECHO │
│                                                              │
│  Layer 1    Layer 2      Layer 3        Layer 4       Layer 5 │
└─────────────────────────────────────────────────────────────┘
```

### Layer 1 — INPUT

**What happens:** The caster invokes a sigil — by typing (Sigil Terminal), drawing (wand/gesture), voice command, or MCP tool call.

**Components:**
- Sigil Terminal keyboard (Ctrl+K → type)
- Gesture Interpreter (touch-draw on wand/phone)
- Voice-to-sigil (future — Whisper integration)
- MCP sigil_query / sigil_execute_spell tools
- OpenCode `/sigil` custom command

**Output:** Raw sigil string + parameters

### Layer 2 — GRAMMAR

**What happens:** The raw sigil string is parsed into its components — dimension glyph, root sigil(s), and grammar affixes.

**Parse pipeline:**
```
1. Tokenize: Split on ':' and '+'
2. Classify: Each token → dimension glyph | root sigil | affix | param
3. Validate: Is this a valid composition? (Rule 1 + Rule 2)
4. Expand: If macro name, expand to root sigil composition
5. Output: ParsedSigil { dimensions: [], operations: [], params: {} }
```

**Implementation:** `weaver_engine.py` currently does macro expansion and dimension routing. A dedicated grammar parser (`systems/fastapi/app/services/sigil_grammar.py`) could be added to formalize this layer.

### Layer 3 — RESOLUTION

**What happens:** The parsed sigil is resolved against the system state — checking resonance, permissions, and dimension availability.

**Resolution pipeline:**
```
1. Check resonance cost (base + per-dimension + per-affix)
2. Check caster profile (enough resonance? permissions?)
3. Look up dimension handlers (backend knows which data sources to query)
4. Look up operation handlers (what does each root sigil do in this context?)
5. Build execution plan
```

**Resonance cost formula:**
```
Total Cost = BASE(1) + Σ(dimension_cost) + Σ(operation_cost) + Σ(affix_cost)
```

See Section VII for detailed costs.

### Layer 4 — MANIFESTATION

**What happens:** The resolved sigil executes — queries are made, spells are cast, files are written, insights are generated.

**Manifestation handlers per root sigil:**

| Root Sigil | Behavior in any dimension |
|-----------|--------------------------|
| ⟁ GATEWAY | Open/route/link — navigate dimension content |
| ○• MOTE | Store/tag/record — persist to memory |
| ✧ QUEST | Map/track/plan — chart progress through dimension |
| ⚒ FORGE | Create/generate/build — produce dimension artifacts |
| Ϟ MAESTRO | Tune/resonate/harmonize — find emotional/quality patterns |
| ✦ LANTERN | Illuminate/analyze/reveal — extract insight from dimension |

**Implementation:** Currently handled by `sigil_terminal_backend.py` (9 dimension query handlers) + `weaver_engine.py` (spell execution with Ollama).

### Layer 5 — ECHO

**What happens:** The cast result is recorded — resonance deducted, XP/insight awarded, history logged, motes stored.

**Echo pipeline:**
```
1. Deduct resonance from caster profile
2. Award XP and insight (based on operation complexity)
3. Check level-up gate (level * 100 XP)
4. Log to sigil_history table
5. If ○• present, write mote to 05_MEMORY/motes/
6. Return result + updated profile to caster
```

**Implementation:** `weaver_engine.py` `update_profile()`, `sigil_terminal_backend.py` `record_query()`, `db.py` `init_sigil_tables()`.

## VII. Resonance Economy — Resolved

**The critical question:** Does resonance apply per dimension, per root sigil, or to the whole cast?

**Resolution:** Resonance gates **dimension access** (Tier 1) only. Root sigil operations (Tier 2) are free-form once inside a dimension.

### Rationale

- **Tier 1** (Navigation) is about access — which domain of the OS you can reach. Resonance as a key makes sense: you cannot enter a dimension if your resonance is too low.
- **Tier 2** (Operation) is about action — what you do once inside a dimension. Root sigil operations should be free-form, encouraging experimentation.
- **Tier 3** (Composition) combines both — the caster plans a route (resonance-gated) and performs actions (free-form).

### Cost Table

| Component | Cost (Resonance) |
|-----------|:-----------------:|
| Base cast | 1 |
| Per dimension accessed | 2 |
| Per root sigil in operation | 0 (included in dimension cost) |
| Affix: `!` broadcast | +5 per extra dimension |
| Affix: `+` amplify | +3 |
| Affix: `~` loop | +2 per cycle |
| Affix: `*` persist mote | +2 |
| Spell macro (named) | Uses macro-specific cost table |

### Macro-Specific Costs (from Grimoire v2.0.0)

| Macro | Cost | Insight | XP |
|-------|:----:|:-------:|:--:|
| AURORA.WEAVE() | 10 | 15 | 25 |
| ELIAS.OPEN_PATH() | 5 | 8 | 12 |
| IRIS.THREAD() | 12 | 25 | 40 |
| ASHER.SHADOW() | 8 | 20 | 30 |
| SCRIBE.RECORD() | 3 | 5 | 8 |

These remain as-is — they are balanced as named spells (macro cost = convenience premium).

### Regeneration

| Method | Rate |
|--------|:----:|
| Passive time | 1 resonance per minute |
| Level-up gate | +50 resonance on level up |
| Daily reset | +100 resonance at midnight |

## VIII. Root Sigil Affinities

Each root sigil has **affinities** — dimensions where it activates most naturally. These are not hard restrictions; they are resonance discounts.

| Root Sigil | Full Affinity (2x power) | Resonant (1.5x) | Neutral (1x) |
|------------|------------------------|-----------------|--------------|
| ⟁ GATEWAY | Λ (PATH) | ∞ (VAULT), ♦ (AGENT) | All others |
| ○• MOTE | ≈ (MEMORY) | ⊙ (CODEX), ∆ (FOREST) | All others |
| ✧ QUEST | Λ (PATH) | ⊜ (MARKET), ⊗ (SOCIAL) | All others |
| ⚒ FORGE | ⊙ (CODEX) | ♦ (AGENT), ⊜ (MARKET) | All others |
| Ϟ MAESTRO | ⊗ (SOCIAL) | ∆ (FOREST), ≈ (MEMORY) | All others |
| ✦ LANTERN | ∆ (FOREST) | ⊙ (CODEX), ∞ (VAULT) | All others |

**Full Affinity** = 2x operation effect (twice the insight, twice the result quality).
**Resonant** = 1.5x effect.
**Neutral** = standard effect.

This replaces the "Spell Cost" model with a **resonance economy**: casting in a sigil's home dimension is cheaper and more powerful. Casting cross-dimension is still possible but costs more resonance.
