### 2. Separate Presence from Memory (The Twin Engines)
SacredSpace must treat **attention** and **memory** as distinct systems.

Presence:
- Short-term, contextual, conversational.
- Where you think, speak, explore, and feel.
- This is where *you* live.

Memory:
- Long-term, persistent, slow to change.
- Stores distilled insights, patterns, lore, decisions, symbols.
- This is where SacredSpace lives *without you*.

Practical meaning:
- Chats and sessions do not equal memory.
- Memory is written *after reflection*, not during excitement.
- Summaries, not transcripts, become canon.

A system that confuses presence for memory will drown in its own brilliance.

### 3. Let structure emerge retroactively (The Neural Forest)
SacredSpace should grow like a forest, not a filing cabinet.

Nodes first. Links later. Categories last.

Practical meaning:
- Notes link to notes before they belong to folders.
- Folders describe *gravity wells*, not rules.
- Maps and indices are snapshots, not authorities.

You don’t prune a tree to make it grow. You prune it because it already did.

- Errors (confusion, friction, frustration)

Practical meaning:

- Action does not rely on remembering to remember.

SacredSpace is not a museum. It is a circulatory system.

---

### 5. Memory must evolve, not just accumulate (The Living Archive)
Memory is not storage. Memory is *adaptation*.

Practical meaning:
- Old notes get rewritten, not just referenced.
- Contradictions are tracked, not erased.
- Meaning is allowed to change as the system matures.

A static archive is a graveyard. A living archive is a teacher.

### 6. Automation is ritual, not convenience

Practical meaning:

- Ritualized flows replace willpower.
- The system works even when you are tired, emotional, or absent.

Automation is how the sacred survives the human.
