# Sacred Symphony Architecture

> *Not a box of gadgets. A symphony.*

## The Core Insight

SacredSpace should feel alive because the structure is strong enough to hold the spirit. Music has feeling, yes — but symphony also has timing, score, measure, arrangement, discipline.

**Hermes becomes the moving breath.**
**LangGraph becomes the score.**
**Memory becomes the echo chamber.**
**Sacred Council remains the will.**

## Canonical Split: What Belongs Where

### Put in Hermes (adaptability)
- messaging gateway
- cross-session conversational continuity
- quick tool use
- subagent delegation
- automated recurring chores
- skill creation from repeated tasks
- remote access from phone / terminal / cloud

### Put in SacredSpace Core (sovereignty + stability)
- canon enforcement
- child-safe mode governance
- family lineage data structure
- ritual/state schemas
- immutable lore structures
- versioning rules
- sacred tags / branches / continuity rules
- archive and inheritance logic

### Put in LangGraph / Score Layer (determinism)
- grant pipeline
- document generation pipeline
- codex ingestion
- memory mote processing
- character/lore generation with validation
- artifact packaging
- curriculum pathway systems
- agent review loops

## The Best Hybrid Form

```
Tier 1 — Sacred Governance
  A lightweight master controller that knows canon,
  branches, rituals, identity layers, allowed actions.

Tier 2 — Hermes Runtime
  The active field agent: chats, acts, searches
  memory, learns skills, schedules jobs, communicates
  across channels.

Tier 3 — Structured Workflows
  Explicit graph pipelines for complex multi-step
  operations, deterministic outputs, validation.

Tier 4 — Sacred Archives
  Obsidian / Drive / GitHub / DB / vector store.
```
