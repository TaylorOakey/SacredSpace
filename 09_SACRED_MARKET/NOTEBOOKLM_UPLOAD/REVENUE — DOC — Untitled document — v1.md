### **1\) `diff_yaml()` “RENAMED” detection is wrong (logic bug)**

Right now:

old\_mapped \= SAFE\_KEY\_MAP.get(k, k)  
if old\_mapped in new and new\[old\_mapped\] \== new.get(k):  
   changes.append(f"    RENAMED:  {k} → {old\_mapped}")

* `new.get(k)` is using the **old key** (which usually won’t exist in `new`)

* So the rename branch almost never triggers; you’ll see “REMOVED/ADDED” instead of “RENAMED”.

**Fix approach:** compare old key → mapped key directly.

def diff\_yaml(old: dict, new: dict) \-\> list:  
   changes \= \[\]  
   for old\_k, old\_v in old.items():  
       new\_k \= SAFE\_KEY\_MAP.get(old\_k, old\_k.lower().replace("-", "\_").replace(" ", "\_"))  
       if old\_k \!= new\_k and new\_k in new and new\[new\_k\] \== old\_v:  
           changes.append(f"    RENAMED:  {old\_k} → {new\_k}")  
       elif old\_k in new and new\[old\_k\] \!= old\_v:  
           changes.append(f"    CHANGED:  {old\_k}: {old\_v}  →  {new\[old\_k\]}")  
       elif old\_k not in new and new\_k not in new:  
           changes.append(f"    REMOVED:  {old\_k}")  
   for k in (set(new.keys()) \- set(old.keys())):  
       \# skip “added” keys that were already counted as renames  
       if any(line.endswith(f"→ {k}") for line in changes):  
           continue  
       changes.append(f"    ADDED:    {k}: {new\[k\]}")  
   return changes

This makes the output *much* more readable.

### **2\) Your frontmatter regex is too strict (will miss valid notes)**

Current regex requires:

* file starts with `---`

* then `\n---\n`

* and exactly one newline layout

Obsidian notes often have:

* Windows line endings (`\r\n`)

* extra whitespace after the closing `---`

* BOM at file start

* frontmatter not at byte 0 because of blank lines

**Fix:** support `\r\n`, allow leading whitespace, and handle notes with frontmatter followed by EOF cleanly.

FRONTMATTER\_RE \= re.compile(  
   r'^\\ufeff?\\s\*---\\s\*\\r?\\n(.\*?)\\r?\\n---\\s\*\\r?\\n?(.\*)$',  
   re.DOTALL  
)

That one change increases your “hit rate” a lot.

### **3\) You claim “rollback-friendly backups” but you only do one zip**

A zip backup is good, but it’s not *rollback-friendly* in the “oops I only want to revert 3 files” sense.

Two upgrades that make rollbacks painless:

* **Per-file backup copy** next to the file (e.g., `note.md.bak`)

* OR write a **migration journal** listing modified files \+ their before/after hashes

Minimal per-file backup:

def backup\_file(path: Path):

   if not bak.exists():  
       shutil.copy2(path, bak)

Call it right before writing during live mode.

### **4\) YAML dumping can distort Obsidian-friendly formatting**

`yaml.dump()` can:

* change quoting

* reformat lists

* emit `null`

* reorder keys depending on emitter behavior

Obsidian mostly doesn’t care, but humans do.

Recommended tweaks:

function Get-SystemReportObject {  
 $os \= Get-CimInstance Win32\_OperatingSystem  
 $cs \= Get-CimInstance Win32\_ComputerSystem  
 $cpu \= Get-CimInstance Win32\_Processor | Select-Object \-First 1  
 $gpu \= Get-CimInstance Win32\_VideoController | Select-Object Name, DriverVersion, AdapterRAM | Select-Object \-First 5  
 $ramGB \= \[Math\]::Round($cs.TotalPhysicalMemory / 1GB, 2\)  
 $disks \= Get-CimInstance Win32\_LogicalDisk \-Filter "DriveType=3" |  
   Select-Object DeviceID, VolumeName, @{n="SizeGB";e={\[math\]::Round($\_.Size/1GB,2)}}, @{n="FreeGB";e={\[math\]::Round($\_.FreeSpace/1GB,2)}}

 $net \= Get-NetAdapter | Where-Object {$\_.Status \-eq "Up"} | Select-Object Name, InterfaceDescription, LinkSpeed, MacAddress

 \[pscustomobject\]@{  
   Timestamp \= (Get-Date).ToString("o")  
   ComputerName \= $env:COMPUTERNAME  
   UserName \= $env:USERNAME  
   IsAdmin \= (Test-IsAdmin)  
   OS \= "$($os.Caption) ($($os.Version)) Build $($os.BuildNumber)"  
   Manufacturer \= $cs.Manufacturer  
   Model \= $cs.Model  
   CPU \= $cpu.Name  
   RAM\_GB \= $ramGB  
   GPU \= $gpu  
   Disks \= $disks  
   NetworkAdaptersUp \= $net  
   PowerShellVersion \= $PSVersionTable.PSVersion.ToString()  
 }  
}

function Write-SystemReportFiles {

 $txtPath  \= Join-Path $OutDir "system\_report.txt"

 $obj | ConvertTo-Json \-Depth 6 | Set-Content \-Path $jsonPath \-Encoding UTF8

 $lines \= @()  
 $lines \+= "Timestamp: $($obj.Timestamp)"  
 $lines \+= "Computer:  $($obj.ComputerName)"  
 $lines \+= "User:      $($obj.UserName)"  
 $lines \+= "Admin:     $($obj.IsAdmin)"  
 $lines \+= "OS:        $($obj.OS)"  
 $lines \+= "Make/Model:$($obj.Manufacturer) / $($obj.Model)"  
 $lines \+= "CPU:       $($obj.CPU)"  
 $lines \+= "RAM (GB):  $($obj.RAM\_GB)"  
 $lines \+= "PS:        $($obj.PowerShellVersion)"  
 $lines \+= ""  
 $lines \+= "GPU:"

 $lines \+= ""

 $lines \+= ""  
 $lines \+= "Network (Up):"  
 foreach ($n in $obj.NetworkAdaptersUp) { $lines \+= "  \- $($n.Name) | $($n.LinkSpeed) | $($n.MacAddress)" }

 $lines | Set-Content \-Path $txtPath \-Encoding UTF8

 Write-Log "Saved system reports to: $OutDir" "OK"  
}

function Winget-Install {  
 param(  
   \[Parameter(Mandatory)\]\[string\]$Id,  
   \[Parameter(Mandatory)\]\[string\]$Name  
 )  
 if (-not (Test-Winget)) { Write-Log "winget unavailable; cannot install $Name." "WARN"; return }

 \# Check if already installed (best-effort)  
 $already \= $false  
 try {  
   $list \= & winget list \--id $Id 2\>$null  
   if ($list \-match $Id) { $already \= $true }  
 } catch { }

 if ($already) {  
   Write-Log "Already installed: $Name ($Id)" "OK"  
   return  
 }

 Write-Log "Installing: $Name ($Id)" "INFO"  
 try {  
   & winget install \--id $Id \--exact \--accept-package-agreements \--accept-source-agreements \--silent  
   Write-Log "Installed: $Name" "OK"  
 } catch {  
   Write-Log "winget install failed for $Name by id. Trying by name search..." "WARN"  
   try {  
     & winget install \--name $Name \--accept-package-agreements \--accept-source-agreements \--silent  
     Write-Log "Installed (by name): $Name" "OK"  
   } catch {  
     Write-Log "FAILED to install: $Name. Error: $($\_.Exception.Message)" "ERR"  
   }  
 }  
}

function Write-SystemReportFiles {  
 param(\[string\]$OutDir)  
 if (-not (Test-Path $OutDir)) { New-Item \-ItemType Directory \-Path $OutDir \-Force | Out-Null }  
 $obj \= Get-SystemReportObject  
 $jsonPath \= Join-Path $OutDir "system\_report.json"  
 $txtPath  \= Join-Path $OutDir "system\_report.txt"  
 $obj | ConvertTo-Json \-Depth 6 | Set-Content \-Path $jsonPath \-Encoding UTF8

 $lines \= @()  
 $lines \+= "Timestamp: $($obj.Timestamp)"  
 $lines \+= "Computer:  $($obj.ComputerName)"  
 $lines \+= "User:      $($obj.UserName)"  
 $lines \+= "Admin:     $($obj.IsAdmin)"  
 $lines \+= "OS:        $($obj.OS)"  
 $lines \+= "Make/Model:$($obj.Manufacturer) / $($obj.Model)"  
 $lines \+= "CPU:       $($obj.CPU)"  
 $lines \+= "RAM (GB):  $($obj.RAM\_GB)"  
 $lines \+= "PS:        $($obj.PowerShellVersion)"  
 $lines \+= ""  
 $lines \+= "GPU:"  
 foreach ($g in $obj.GPU) { $lines \+= "  \- $($g.Name) | Driver $($g.DriverVersion)" }  
 $lines \+= ""  
 $lines \+= "Disks:"  
 foreach ($d in $obj.Disks) { $lines \+= "  \- $($d.DeviceID) $($d.VolumeName) | Size $($d.SizeGB)GB | Free $($d.FreeGB)GB" }  
 $lines \+= ""  
 $lines \+= "Network (Up):"  
 foreach ($n in $obj.NetworkAdaptersUp) { $lines \+= "  \- $($n.Name) | $($n.LinkSpeed) | $($n.MacAddress)" }

 $lines | Set-Content \-Path $txtPath \-Encoding UTF8  
 Write-Log "Saved system reports to: $OutDir" "OK"  
}

function Winget-Install { param(\[string\]$Id,\[string\]$Name)  
 if (-not (Test-Winget)) { Write-Log "winget unavailable; cannot install $Name." "WARN"; return }  
 $already \= $false  
 try { $list \= & winget list \--id $Id 2\>$null; if ($list \-match $Id) { $already \= $true } } catch {}  
 if ($already) { Write-Log "Already installed: $Name ($Id)" "OK"; return }

 try { & winget install \--id $Id \--exact \--accept-package-agreements \--accept-source-agreements \--silent; Write-Log "Installed: $Name" "OK" }  
 catch { Write-Log "FAILED to install: $Name. $($\_.Exception.Message)" "ERR" }

## **3\) Can you use a Cloud Shell interface?**

**Yes — but it’s for “cloud things,” not “local laptop things.”** ☁️🧰

### **✅ Cloud Shell is great for:**

* running cloud CLIs (AWS / Azure / GCP)

* Terraform deployments

* quick scripts against cloud resources

* Git operations against cloud repos

Examples:

* **AWS CloudShell** is a browser-based pre-authenticated shell; it’s meant for interactive workloads and has session/storage limits.

* **Google Cloud Shell** is a temporary VM with a **persistent home directory** (5GB) for your user files.

### **❌ Cloud Shell is *not* good for:**

* managing your **local Docker Desktop**

* touching your **local SacredSpace folders** directly

* USB/ADB/NFC experiments

* LAN routing experiments on your physical devices

That’s because Cloud Shell lives in the provider’s cloud environment (not your laptop), and it’s optimized for interactive cloud ops.

### **Best practice (your use-case):**

* Use **local PowerShell / Windows Terminal / WSL** for your **Portal \+ Edge Stack \+ SacredSpace OS**

* Use **Cloud Shell** only when you start deploying parts of this stack to a cloud VPS / AWS / Azure / GCP
